ENCRYPTION_SECRET Implementation Summary
========================================

Changes Made:
-------------

1. modules/envs/locals.tf
   - Added random_password resource to generate 32-char encryption secret per environment

2. modules/envs/outputs.tf
   - Added encryption_secret output (marked sensitive)

3. ssm-backend-secrets.tf
   - Added EncryptionSecret parameter to SSM document
   - Added ENCRYPTION_SECRET shell variable
   - Added --from-literal=ENCRYPTION_SECRET to kubectl command
   - Added EncryptionSecret to association parameters

Result:
-------
Each environment gets a unique, randomly-generated 32-character encryption secret
that is automatically injected as the ENCRYPTION_SECRET environment variable
in the vytalmed-backend-secrets Kubernetes secret.

The backend application can now read process.env.ENCRYPTION_SECRET to encrypt
sensitive data before storing in the database.

To Apply:
---------
terraform apply

The encryption secret will be generated and stored in Terraform state (encrypted
if using remote state with encryption enabled).
